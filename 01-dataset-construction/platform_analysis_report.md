# 电商平台数据采集方案分析报告

> PVTT (Product Video Template Transfer) 项目 — 数据集构建
> 日期: 2026-03-13 (更新: 深度调研8平台)

## 一、项目需求

| 维度 | 要求 |
|------|------|
| **目标数据** | 商品展示视频 + 商品图片 |
| **品类** | 珠宝首饰（necklace, bracelet, earring, ring）、配饰（sunglasses, watch, handbag） |
| **视频要求** | 商品被佩戴/使用/展示的视频，用于模板迁移 |
| **数据规模** | 目标 1000+ 产品，500+ 视频 |
| **预算** | 学术项目，需控制成本 |

---

## 二、各平台综合评估

### 评估维度说明
- **爬取难度**: ★越多越难（1-5星）
- **视频丰富度**: 该平台商品带视频的比例
- **数据质量**: 视频分辨率、图片清晰度、元数据完整性
- **成本**: 免费爬取 / API费用 / 第三方服务费用

---

### 2.1 Amazon（亚马逊）⭐ 推荐

| 维度 | 评价 |
|------|------|
| **爬取难度** | ★★☆☆☆（中低） |
| **反爬机制** | 自定义WAF，频率限制，IP封禁 |
| **可行方法** | requests + 住宅IP（已验证可行） |
| **视频丰富度** | ★★★★☆（~40-60% 商品有视频） |
| **视频质量** | 720p-1080p MP4，时长5-30秒 |
| **图片质量** | 高分辨率（1500x1500+），多角度 |
| **元数据** | 价格、标题、品牌、评分、分类 — 非常完整 |

**当前数据**: 404产品, 1888图片, 378视频, 2.8GB

**成本分析**:
- 免费自行爬取: ✅ 可行（需住宅IP，不可在服务器运行）
- Amazon PA-API: 免费但将于2026年4月停用，迁移至Creators API
- Apify Amazon Scraper: 免费额度$5/月（约50-100个产品），$29/月起
- Bright Data: $1.50/1000请求（约$7.5爬5000个产品页）

**结论**: **最佳选择**。已有成熟爬虫，免费，视频丰富度高。继续扩展。

### 2.2 Etsy

| 维度 | 评价 |
|------|------|
| **爬取难度** | ★★★★★（极难） |
| **反爬机制** | **Datadome** — CAPTCHA拦截所有自动化（含真实Chrome） |
| **可行方法** | 官方API（仅图片）/ Apify付费爬虫 |
| **视频丰富度** | ★★☆☆☆（~10-15% 商品有视频） |
| **视频质量** | 720p MP4 |
| **图片质量** | 高分辨率（il_1588xN），手工艺品风格独特 |
| **元数据** | 价格、标题、卖家、材质 — 较完整 |

**成本分析**:
- 免费爬取: ❌ 不可行（Datadome完全封锁）
- Etsy Open API v3: **免费**注册，可获取listing图片，**不支持视频**，限5个shop
- Apify Etsy Scraper: 免费额度可爬约50个产品，$29/月起可规模化
- Bright Data Etsy Scraper: $2.50/1000请求
- Oxylabs: $49/月起

**结论**: 图片可通过免费API获取，视频需付费服务。性价比一般。

### 2.3 eBay

| 维度 | 评价 |
|------|------|
| **爬取难度** | ★★★☆☆（中等） |
| **反爬机制** | JS渲染 + 频率限制，无强CAPTCHA |
| **可行方法** | 官方Browse API（免费）/ curl_cffi可获取部分数据 |

| **视频质量** | 中等，卖家自拍质量参差不齐 |
| **图片质量** | 中等（卖家上传，非标准化） |
| **元数据** | 价格、标题、卖家、竞拍信息 — 较完整 |

**成本分析**:
- 免费爬取: ⚠️ 部分可行（搜索页JS渲染需浏览器）
- eBay Browse API: **免费**注册开发者账号，支持搜索、商品详情、图片
- eBay Media API: 免费，支持视频上传/获取
- Bright Data: $1.50/1000请求

**结论**: API免费且支持图片+视频，但珠宝类视频少，图片质量不稳定。

### 2.8 淘宝/天猫

| 维度 | 评价 |
|------|------|
| **爬取难度** | ★★★★★（极难） |
| **反爬机制** | 行为指纹 + AI风控 + 动态签名 + Cookie 10分钟过期 |
| **可行方法** | 需Selenium/DrissionPage + 登录态 + 代理池 |
| **视频丰富度** | ★★★★★（~70%+，主图视频几乎标配） |
| **视频质量** | 高（1080p，专业制作，模特展示） |
| **图片质量** | 高（标准化白底图 + 场景图） |
| **元数据** | 极其丰富（价格、销量、评价、规格） |

**成本分析**:
- 免费爬取: ❌ 极其困难（2025年风控升级，IP高频检测阈值极低）
- 淘宝开放平台API: 需企业资质申请，个人开发者基本无法获取
- 第三方数据服务: ¥0.01-0.05/条（批量采购）
- Bright Data: 不直接支持

**结论**: 视频质量最高、最丰富，但爬取难度也最高。如预算允许（¥100-500可获取大量数据），第三方数据服务是最佳路径。

### 关键注意事项

1. **学术合规**: 论文中注明数据来源，仅用于学术研究目的（参考SAGE 2025论文最佳实践）
2. **数据多样性**: 跨平台数据可展示模型泛化能力（reviewer加分项）
3. **视频质量筛选**: 需后处理过滤低质量视频（模糊、过短、无产品展示）

5. **中国平台**: 避免直接爬取，通过合规第三方服务采购数据
6. **视频格式**: TikTok竖屏(9:16)视频需后处理裁剪/调整；淘宝/京东横屏(16:9)视频最适合PVTT

## A1. 淘宝/天猫 (Taobao/Tmall)

### 1. 爬取可行性

**难度: 极难 (5/5)**

淘宝是中国反爬技术最先进的平台之一。2025-2026年主要反爬措施包括:

- **行为指纹分析**: 追踪鼠标轨迹、滚动模式、点击间隔，AI模型实时判断是否为自动化操作
- **动态签名验证**: Cookie有效期仅约10分钟，API请求需要动态签名，签名算法频繁更新
- **IP高频检测**: 阈值极低，数据中心IP直接封禁，住宅IP频繁请求也会触发验证
- **滑块/图形验证码**: 触发率极高，需要高级验证码解决方案
- **登录门控**: 多数商品详情页和视频需要登录才能访问

## A2. Etsy

### 1. 爬取可行性

**难度: 极难 (5/5)**

Etsy采用了业界最激进的反爬方案之一 — **DataDome**:

- **TLS指纹检测**: 分析TLS握手特征，自动化工具的TLS指纹与真实浏览器不同
- **行为分析**: 追踪鼠标抖动(jitter)、滚动模式、浏览节奏、事件时间戳
- **浏览器环境检测**: 检测Navigator属性、Canvas/WebGL指纹
- **请求模式分析**: 并发请求数、请求间隔均匀度
- **地域限制**: 基于IP地理位置限制数据访问
- **持续更新**: DataDome与Google合作，保护方案持续升级

**工具测试结果** (已实测):
- `requests` + headers伪装: 403 Forbidden
- `curl_cffi` (模拟Chrome TLS): 403 Forbidden
- `DrissionPage` (无头Chrome): CAPTCHA拦截
- `DrissionPage` (有头Chrome): 仍被CAPTCHA拦截
- 结论: **所有自动化方法均被封锁**

## A8. Shopee（虾皮）

## 法律合规总结 (2025-2026)

### 美国法律框架
- **公开数据爬取**: 一般合法（hiQ v. LinkedIn先例）
- **Fair Use**: 学术研究通常符合四因素测试中的第一项（非商业/学术目的）
- **CFAA**: 不绕过技术保护措施（如不破解登录门控）
- **建议**: 仅爬取公开可访问数据，不绕过认证

### 欧盟法律框架
- **2019指令**: 允许研究机构进行文本和数据挖掘
- **GDPR**: 即使是嵌入在公开评论中的个人数据也需要合法依据

### 中国法律框架
- **《网络安全法》《数据安全法》《个人信息保护法》**: 三法联防，保护严格
- **学术豁免**: 不如美国Fair Use清晰
- **建议**: 中国平台数据通过合规第三方采购，避免直接爬取

### 学术研究最佳实践 (参考: SAGE论文 "Web scraping for research" 2025)
1. 仅采集公开数据
2. 不绕过登录或技术保护
3. 尊重robots.txt和API速率限制
4. 论文中明确标注数据来源和用途
5. 不采集/存储个人身份信息
6. 优先使用官方API而非爬取
